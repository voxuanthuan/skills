# Interview

Read the file specified in `$ARGUMENTS` and interview the user in detail using the AskUserQuestion tool about literally anything: technical implementation, UI & UX, concerns, tradeoffs, etc. 

Make sure the questions are not obvious. Be very in-depth and continue interviewing continually until it's complete, then write the spec to the file.

## Guidelines

- Ask about technical implementation details
- Cover UI/UX considerations
- Explore concerns and edge cases
- Discuss tradeoffs and decisions
- Avoid obvious questions
- Continue interviewing until comprehensive
- Write the final spec back to the file when complete
